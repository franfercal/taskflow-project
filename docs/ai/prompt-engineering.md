## Prompt útilies

1. Clean Code y buenas prácticas:

    "Tengo este código en Javascript que funciona, pero es difícil de leer y mantener: [insertar código]. Por favor, refactorízalo siguiendo los principios SOLID. Extrae las responsabilidades en funciones más pequeñas y añade comentarios explicativos breves."

2. Entender código:

    "Acabo de leer este proyecto y no entiendo qué hace exactamente este script de Javascript: [insertar código]. Explícamelo línea por línea como si se lo estuvieras enseñando a un desarrollador junior."

3. Generación automática de documentación:

    "Escribe un archivo README.md estructurado y profesional para mi nuevo proyecto de código abierto. El proyecto es un [descripción del proyecto]. Incluye secciones para: Requisitos previos, Instalación, Uso de ejemplo y Cómo contribuir."

4. Creación de pruebas unitarias:

    "Escribe pruebas unitarias usando Jest y Puppeteer. Asegúrate de simular los clics de los usuarios.

5. Expresiones Regulares (Regex) y utilidades:

    "Escribe una expresión regular en JavaScript para validar un campo de contraseña. Debe tener al menos 8 caracteres, incluir una letra mayúscula, un número y un símbolo especial (@, $, !, %, *, ?, &). Explícame cómo funciona cada parte de la regex generada."

6. Componentes UI específicos:

    "Actúa como un desarrollador frontend experto. Escribe el código para [insertar caso] (usando [insertar tecnología a usar]) que funcione como [insertar funcionalidad]. Debe [insertar el comportamiento que debe tener]"

7. Análisis de errores de consola:

    "Tengo el siguiente error en mi aplicación [insertar framework o lenguaje]: [insertar texto exacto del error]. Aquí está el fragmento de código donde ocurre: [insertar código]. ¿Puedes explicarme por qué ocurre este error y darme la solución paso a paso para arreglarlo?"

## Funcionamiento

Un prompt debe de ser específico, claro, contextualizado y estructurado. Por eso los anteriores funcionan bien.
